Abstract

The ElementTraversal specification defines a small set of shortcut accessors used while navigating the structure (i.e. elements) of a DOM. While it does not expose information previously unavailable, it has been designed to help authors avoid mistakes commonly found programs that manipulate the DOM and is well-suited for implementation in mobile devices where it may be impossible to support the entirety of the DOM.

2. The ElementTraversal interface

All instances of the Element node type MUST support the ElementTraversal interface as well.

The ElementTraversal interface

interface ElementTraversal {

  readonly attribute Element  firstElementChild;
  readonly attribute Element  lastElementChild;
  readonly attribute Element  nextElementSibling;
  readonly attribute Element  previousElementSibling;
};

Attributes

firstElementChild of type Element, readonly

Returns the first child element node of this element, or null if this element has no child elements.

lastElementChild of type Element, readonly

Returns the last child element node of this element, or null if this element has no child elements.

nextElementSibling of type Element, readonly

Returns the next sibling element node of this element, or null if this element has no following sibling element node.

previousElementSibling of type Element, readonly

Returns the previous sibling element node of this element, or null if this element has no preceding sibling element node.
